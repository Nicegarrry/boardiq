# BoardIQ — Design Brief

## Identity

BoardIQ is a board governance platform that replaces the 500-page board pack with an AI-powered intelligence briefing. The audience is board directors, CEOs, CFOs, and company secretaries — people who value precision, restraint, and trust over novelty.

The interface should feel like it was designed by a governance consultancy's in-house design team, not generated by an AI tool.

---

## Anti-AI Design Principles

Every design choice should pass this test: *would a thoughtful human designer make this choice for this specific context, or is this a pattern that gets generated by default?*

**Avoid:**
- Traffic-light status colors (red/amber/green chips). These are the single most generic pattern in dashboard design. Severity and status must be communicated through form and weight, not just hue.
- Rounded pill badges with saturated fills. These scream "component library default."
- Blue-purple gradient anything. The most overused AI aesthetic.
- Uniform card grids with identical sizing. Real editorial layouts have hierarchy.
- Icon-heavy interfaces. Text communicates governance concepts better than pictograms.
- Drop shadows on everything. Use borders and background shifts for depth.
- Emoji in UI chrome. Never.
- Perfectly symmetrical layouts. Slight asymmetry signals intentional design.
- Sans-serif-only typography. Serif type signals authority and editorial intent.

**Instead:**
- Use typographic weight and density to signal importance. A bold label does more work than a colored chip.
- Use the accent color sparingly — it should feel like a hand-placed editorial highlight, not a system default.
- Use whitespace aggressively. Governance content is dense; the UI must create breathing room.
- Use borders and subtle background tinting rather than colored fills for status.
- Let content hierarchy emerge from type size, weight, and position — not from decoration.

## Color System

### Primary Palette

```
--ember:          #C2410C    Hero accent. Used for: active states, primary buttons,
                              the BoardIQ logo accent, key interactive elements.
                              Appears in ~5% of any given screen.

--ember-muted:    #9A3412    Text on ember-tinted backgrounds.
--ember-tint:     #FFF7ED    Subtle background tint for ember-related elements.
--ember-border:   #FDBA74    Border for ember-tinted containers.
```

### Neutral Palette (warm greyscale)

These are warm-shifted, not blue-grey. The warmth is subtle but essential — it gives the interface a printed-paper quality.

```
--paper:          #FAF9F7    Primary background. Warm off-white.
--paper-dark:     #F5F3F0    Secondary background. Inset areas, table headers.
--surface:        #FFFFFF    Card surfaces.
--border:         #E8E5E0    Primary borders. Warm grey.
--border-light:   #F0EEEA    Subtle dividers within cards.

--ink:            #1A1A1A    Primary text. Near-black, not pure black.
--ink-secondary:  #5C5C5C    Body text, descriptions.
--ink-muted:      #8C8C8C    Metadata, timestamps, supporting text.
--ink-faint:      #B3B3B3    Disabled states, decorative text.
```

### Status Communication (NOT traffic lights)

Status is communicated through **typographic treatment and border weight**, not background color fills. This is the key anti-AI decision.

```
IQ Severity Levels:
  Alert    → Left border: 2px solid #1A1A1A (ink). Label set in bold uppercase.
             The weight and darkness of the border IS the signal.
  Watch    → Left border: 2px solid #8C8C8C (ink-muted). Label in medium weight.
  Ready    → Left border: 2px solid var(--ember). Label in regular weight.
  Clear    → No left border. Label in muted text.

Paper/Action Status:
  Overdue      → Text in --ink, bold. Small "overdue" label in caps.
  In Progress  → Text in --ink-secondary, medium weight.
  Complete     → Text in --ink-muted, regular weight. Checkmark character (✓) not icon.
  Not Started  → Text in --ink-faint.

Document Read Status:
  Unread  → Filename in --ink (bold, stands out).
  Read    → Filename in --ink-muted (recedes). Subtle ✓ prefix.
```

The principle: **important things are dark and heavy, resolved things are light and faded.** This is how a printed document would work — you bold what matters, not what's fine.

### Action Type Indicators

Instead of colored chips, action types use **typographic labels with subtle structural markers:**

```
DECISION  → Uppercase, bold, --ink. Underscore or left-bar in --ember.
            This is the only action type that gets the accent color.
DISCUSSION → Uppercase, medium weight, --ink-secondary.
APPROVAL   → Uppercase, medium weight, --ink-secondary.
NOTING     → Uppercase, regular weight, --ink-muted.
```

Decisions get ember because they require the director's active participation. Everything else is greyscale.

### IQ Badge Rethink

Instead of colored pills, IQ indicators use an **inline text treatment:**

```
Before (generic):
  [🔴 IQ: Evidence gaps]     ← colored chip with dot

After (editorial):
  IQ — Evidence gaps found    ← plain text, "IQ" in ember, rest in ink-secondary
                                 or
  ⬡ Revenue assumption at risk  ← geometric marker, text only
```

The IQ marker should feel like a marginal note in a well-edited document, not a notification badge from a mobile app.

## Typography

### Font Pairing

```
Display / Headings:   Instrument Serif (400, 400 italic)
                      Used for: meeting date, card titles, section content,
                      the "Board" in BoardIQ logo.
                      Character: authoritative, editorial, warm.

UI / Body:            Hanken Grotesk (300, 400, 500, 600, 700)
                      Used for: labels, metadata, body text, buttons,
                      navigation, form elements.
                      Character: precise, modern, legible at small sizes.
```

### Type Scale

```
Meeting date (hero):     38px / Instrument Serif / 400 / -0.5 tracking
Card title (primary):    20px / Instrument Serif / 400
Card title (secondary):  16px / Instrument Serif / 400
Section label:           12px / Hanken Grotesk / 700 / 1.0 tracking / uppercase
Body text:               13px / Hanken Grotesk / 400 / 1.65 line-height
Metadata:                12px / Hanken Grotesk / 400
Small label:             11px / Hanken Grotesk / 500
Micro label:             10px / Hanken Grotesk / 700 / 1.5 tracking / uppercase
Monospace (item numbers): 11px / system monospace
```

### Logo Treatment

"Board" in Instrument Serif 400 (ink), "IQ" in Instrument Serif 400 italic (ember). No icon, no symbol. The italic shift IS the mark.

## Spatial System

### Spacing Scale

```
4px   — micro gap (within a badge, between icon and text)
8px   — tight gap (between related elements)
12px  — standard gap (between cards, between sections of a card)
16px  — comfortable gap (card internal padding, section spacing)
20px  — generous gap (card padding on important cards)
24px  — section gap
32px  — major section break
```

### Card System

```
Standard card:
  background: var(--surface)
  border: 1px solid var(--border)
  border-radius: 8px
  padding: 16px 20px
  shadow: none (use border, not shadow)

Expanded card:
  Same as standard, plus:
  border-color shifts to var(--border) with slightly more contrast on hover

Briefing card (AI summary):
  Same as standard, plus:
  Top edge: 2px solid var(--ember) as a subtle accent bar
  Slight background tint on the label area

No cards should have:
  - Drop shadows
  - Colored borders (except the briefing card accent bar)
  - Rounded corners > 8px
  - Gradient backgrounds
```

### Layout

```
Max content width:    800px (narrow, editorial feel — like a column of text)
Page padding:         24px horizontal, 36px top
Card gap:             10px (tight, stacked)
Two-column grid:      Used for lower-priority items only. 10px gap.
```

## Interaction Patterns

### Hover States

Cards: border shifts from `--border` to slightly darker. No color change, no shadow. Cursor: pointer.

Buttons: primary (ember) darkens slightly. Secondary (outline) gets a subtle background tint.

Links/interactive text: underline appears on hover. No color change.

### Transitions

All transitions: 150ms ease. No bounce, no spring, no overshoot. Governance software doesn't bounce.

### Expand/Collapse

Cards expand inline with content push (no animation on the expand itself — content simply appears). The border-top divider between collapsed header and expanded content provides the structural signal.

### Slide-Over Panel

Right-aligned, 540px wide. Semi-transparent backdrop (rgba(0,0,0,0.12) — very subtle). No blur effect. Panel has a left border shadow only (no all-around glow).

---

## Iconography

Minimal. Prefer text labels over icons. Where icons are necessary:

- Use stroke-style, 1.5px weight, in `--ink-muted`
- Notification bell, search, and close (×) are the only icons in the top bar
- No icons in cards or content areas — use typographic markers instead
- The IQ indicator uses a small geometric mark (diamond ◆ or hexagon ⬡), not a brain/sparkle/lightbulb

---

## What This Design Is NOT

- It is not playful or whimsical
- It is not dark mode (directors read in well-lit rooms and on planes)
- It is not information-sparse (directors handle dense content professionally)
- It is not visually noisy (every element earns its place)
- It is not a Notion/Linear clone (those are productivity tools; this is a governance tool)
- It is not a dashboard with charts (it's a briefing document with intelligence)
